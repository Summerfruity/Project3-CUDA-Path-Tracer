# BVH 详解：是什么，以及为什么能加速

---

## 一、BVH 是什么？

**BVH** = **Bounding Volume Hierarchy**（层次包围盒）

用最简单的话说：

> **BVH 是一棵二叉树，每个节点代表一个"盒子"，这个盒子刚好包住下面的一部分三角形。从根节点到叶子节点，盒子越来越小，叶子节点里直接存若干原始三角形。**

### 直观的类比

想象你要在一本很厚的词典里找一个单词：

- **暴力做法**：从第1页逐页翻到最后一页，一页一页检查。**O(n)**
- **BVH 做法**：先查目录（A-Z 分区），发现单词以 M 开头，直接跳到 M 区；在 M 区里再查二级目录（Ma, Mb, Mc...），逐步缩小范围，最终只翻了十几页就找到了。**O(log n)**

BVH 就是给三维场景中的三角形建了一套**"空间目录"**。

## 二、BVH 的树结构

```
                    [根节点]
                AABB(整个场景)
               /              \
        [左子树]                [右子树]
    AABB(左半空间)           AABB(右半空间)
      /      \                /       \
  [叶子]    [内部]        [内部]     [叶子]
  4个三角   继续分          继续分     3个三角
```

### 两种节点

| 节点类型 | 存储内容 |
|---------|---------|
| **内部节点 (Internal Node)** | 一个 **AABB**（轴对齐包围盒）+ 指向左右两个子节点的索引 |
| **叶子节点 (Leaf Node)** | 一个 **AABB** + 若干（如 2~8 个）**原始三角形** |

### AABB 是什么？

**Axis-Aligned Bounding Box** = 轴对齐包围盒

就是一个与 XYZ 坐标轴平行的长方体，用 `min(x,y,z)` 和 `max(x,y,z)` 两个角点就能完全定义。

```
        max
         ┌────────┐
        /│       /│
       / │      / │
      ┌──┼─────┐  │
      │  └─────┼──┘
      │ /      │ /
      │/       │/
      └────────┘
     min
```

## 三、BVH 怎么"找交点"？

当一条光线射入场景时，遍历过程如下：

```
1. 测试光线 vs 根节点 AABB
   └─ 没命中？直接返回，整个场景都没交点 ✓
   └─ 命中了？继续...

2. 测试光线 vs 左子节点 AABB
   └─ 命中？继续深入...

3. 测试光线 vs 右子节点 AABB  
   └─ 没命中？整棵右子树直接跳过！✓✓✓

4. 到达叶子节点
   └─ 与该叶子内的少量三角形逐一求交
```

### 核心操作：AABB vs 光线

光线与 AABB 的求交叫做 **Slab Method**，只需要几次加减乘除：

```cpp
// 对 X/Y/Z 三个轴分别算进入/离开距离
t_enter = max( (min - origin) / direction )
t_exit  = min( (max - origin) / direction )

if t_enter < t_exit 且 t_exit > 0:
    光线穿过盒子！
else:
    没穿过，跳过整棵子树
```

**这个测试极快**，远比与单个三角形做 Möller-Trumbore 相交要快得多。

## 四、为什么 BVH 能加速？

### 4.1 核心原因：大量三角形被"成批跳过"

假设一个 Mesh 有 **100,000 个三角形**：

| 方法 | 最坏情况 | 说明 |
|------|---------|------|
| **暴力遍历** | **100,000 次** 三角形测试 | 每条光线测所有面 |
| **BVH 遍历** | **~30 次 AABB 测试 + 几十个三角形测试** | 只对可能命中的子树深入 |

**加速比来自"剪枝"（Pruning）**：

```
根节点 AABB ──┬── 左子树 AABB ──┬── 左子树 AABB ── 叶子(4个三角)
             │                  └── 右子树 AABB ── 没命中！整棵跳过 ✓
             │
             └── 右子树 AABB ── 没命中！跳过 50,000 个三角 ✓✓✓
```

一个 AABB 测试就能淘汰掉子树下成千上万的几何体，这就是**指数级剪枝**。

### 4.2 复杂度对比

| 方法 | 时间复杂度 | 空间复杂度 |
|------|----------|-----------|
| 暴力遍历 | **O(n)** — 与三角形总数成正比 | O(n) |
| BVH | **O(log n)** — 树的高度 | O(n) |

n = 100,000 时：
- log₂(100000) ≈ **17**
- 也就是说最多只需要访问十几个节点！

### 4.3 具体到本项目

你当前 `computeIntersections` 中的代码：

```cpp
for (int i = 0; i < meshRanges_size; i++) {
    // AABB 粗剔（已经做了一层优化）
    for (int j = 0; j < range.triCount; j++) {
        // 每个三角形都测！O(n)
        triangleIntersectionTest(...)
    }
}
```

**问题**：如果通过 AABB 的 Mesh 有 **50,000 个三角形**，那这条光线还是要测 **50,000 次**！

加了 BVH 后：

```cpp
for (int i = 0; i < meshRanges_size; i++) {
    // AABB 粗剔
    if (enableBVH && range.bvhRootIndex >= 0) {
        bvhIntersect(...);  // 只访问 O(log n) 个节点！
    } else {
        // 小 mesh fallback
    }
}
```

即使是 50,000 个面的大 Mesh，BVH 也只会深入 **~20 个节点**，每个节点做极快的 AABB 测试，只在最终叶子节点里测 **2~8 个三角形**。

## 五、画个图来理解

假设一个二维场景（简化理解），有 8 个物体：

```
暴力做法                    BVH 做法
─────────                  ─────────
☀️→ [1][2][3][4][5][6][7][8]    ☀️→ [   AABB(全部)   ]
      测1 测2 测3 测4 测5...           /              \
                                   AABB(左半)      AABB(右半)
                                   /        \      /        \
                                 AABB      AABB  AABB      AABB
                                 /  \      /  \  /  \      /  \
                               [1][2] [3][4]  [5][6] [7][8]
```

**光线只往右上方飞**：

- **暴力**：测了 8 个物体，发现只有 [7][8] 可能命中，前面 6 次都白算了。
- **BVH**：
  1. 测根 AABB — 命中
  2. 测左半 AABB — 没命中！**跳过 [1][2][3][4] 4个物体** ✓
  3. 测右半 AABB — 命中
  4. 测右半的左子 AABB — 没命中！**跳过 [5][6]** ✓
  5. 测右半的右子 AABB — 命中，进叶子测 [7][8]

**结果**：只做了 **4 次 AABB 测试 + 2 次物体测试**，比暴力快 4 倍。如果物体是 10,000 个而不是 8 个，加速比会达到 **数百倍**。

## 六、BVH 的构建思想

BVH 的加速效果取决于**树的质量**。好的树能让不相交的子树尽早被剪掉。

### 常用构建策略

| 策略 | 原理 | 特点 |
|------|------|------|
| **中点分割 (Median Split)** | 选最长轴，按三角形中心坐标排序，取中位数一分为二 | 实现简单，效果够用 |
| **SAH (Surface Area Heuristic)** | 评估不同分割位置的"代价" = 左子树面积×左数量 + 右子树面积×右数量 | 树质量最优，实现稍复杂 |

**SAH 的直觉**：

> 如果一个分割能让左盒子又小又少、右盒子又小又少，那光线"误闯"到不该去的子树的概率就低，剪枝效率就高。

代码用的是**中点分割**，对课设来说已经完全足够。如果想追求极限性能，可以升级为 SAH。

---

## 七、总结

| 问题 | 答案 |
|------|------|
| **BVH 是什么？** | 一棵二叉树，每个节点存一个包围盒，叶子存原始三角形 |
| **为什么快？** | 用极快的 **AABB 测试** 成批淘汰大量不相交的几何体，从 **O(n)** 降到 **O(log n)** |
| **代价是什么？** | 需要额外内存存储树节点（约 2n 个节点），以及一次性的 CPU 建树时间 |
| **GPU 上怎么实现？** | 不能用递归！必须用**显式栈**做迭代遍历，每个线程维护自己的栈 |
